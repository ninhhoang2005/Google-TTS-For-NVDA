Google TTS For NVDA

Developers: Nguyen Anh Duc, Dao Duc Trung and Pham Hung Vuong.

Google TTS For NVDA uses Google's WASM TTS engine locally through a managed headless browser runtime: Microsoft Edge or Google Chrome.

This add-on grew from a simple dream: making Google TTS usable as a practical, everyday NVDA synthesizer on Windows computers.

The add-on exposes Google TTS voices as an NVDA synthesizer and includes a Tools menu Voice Manager for downloading or removing voice packages listed in the bundled voices.json catalog.

Voice packages that are known not to work with the bundled WASM TTS engine are marked as not supported in Voice Manager and are not exposed as synthesizer voices.

Settings include voice, rate, rate boost, pitch, volume, and browser runtime. Voice packages are downloaded only from the Google TTS Voice Manager.

Repeated short phrases under 5000 characters are cached in RAM for the current NVDA session only. The cache is volatile and is cleared automatically when NVDA exits or the computer restarts.

For translation work, Poedit can edit a locale and save synchronized .po and .mo files; in that workflow, build_i18n.py can be used to validate the translation. If another translation tool edits .po files without generating or synchronizing .mo, the i18n script can build the generated translation files.
